Feature Creep – Discussion Answers

1. Rank features by usage (which personas actually use them?)

Let’s assume 3 simple personas for your project:
• Ananya (Student, beginner) → wants quick outfit suggestions
• Riya (Working professional) → wants efficient, occasion-based outfits
• Kabir (Fashion enthusiast) → wants more control & customization

Feature Ranking (Most → Least Used)
1. Outfit Recommendation System → Used by all personas
2. Preference Selection → Used by all personas
3. Outfit Display Interface → Used by all personas
4. Filtering Options → Used by Riya & Kabir
5. Weather-Based Suggestions → Used by Ananya & Riya
6. Save Outfit Feature → Used by Riya & Kabir
7. Basic User Profile → Mostly Kabir
8. Advanced customization (if added) → Only Kabir

Potential Feature Creep
• Advanced customization
• Social sharing
• Complex profile system

👉 These are not essential for most users, so they risk becoming feature creep.

⸻

2. Where is the main tension in your project?

Main tension: Automation vs Control
• Some users want:
• “Just tell me what to wear” (high automation – Ananya)
• Others want:
• “Let me tweak styles, colors, combinations” (high control – Kabir)

Concrete Example
• If the system auto-generates outfits instantly → simple but less flexible
• If you add too many filters/options → powerful but confusing

👉 Your solution:
• Default: simple auto-suggestions
• Optional: “Customize” button for advanced users

⸻

3. How to avoid feature creep?

Your team can follow a simple process:

Step 1: Feature evaluation checklist

Before adding any feature, ask:
• Does it solve a common user need?
• Is it used by at least 2 personas?
• Is it essential for MVP?

⸻

Step 2: Use MVP approach
• Build only core features first
• Delay “nice-to-have” features

⸻

Step 3: Prioritization method

Use:
• Must-have
• Should-have
• Nice-to-have

👉 Only implement Must-have + some Should-have

⸻

Step 4: Say “No” with reason

Example:

“We are not adding social sharing now because it does not support the core goal of outfit recommendation.”
